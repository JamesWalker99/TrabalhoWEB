Painel de Expansão

< mat-expansion-panel> fornece uma visualização de resumo/detalhes expansível.

Conteúdo do painel de expansão

Cada painel de expansão deve incluir um cabeçalho e incluir opcionalmente uma barra de ação.

Personal data Type your name and age account_circle Destination Type the country name map Day of the trip Inform the date you wish to travel date_range

Cabeçalho

A tag < mat-expansion-panel-header> mostra um resumo do conteúdo do painel e atua como o controle para a expansão e recolhimento. Este cabeçalho pode conter, opcionalmente, um < mat-panel-title> e um < mat-panel-description> , que formatam o conteúdo do cabeçalho para alinhar com as especificações do Material Design.

Por padrão, o cabeçalho do painel de expansão inclui um ícone de alternância no final do cabeçalho para indicar o estado de expansão. Este ícone pode ser escondido através da propriedade hideToggle.


Barra de ação

As ações podem, opcionalmente, ser incluídas na parte inferior do painel, visíveis apenas quando a expansão estiver em seu estado expandido.


Desativando um painel

Os painéis de expansão podem ser desativados usando o atributo disable. Um painel de expansão desativado não pode ser alternado pelo usuário, mas ainda pode ser manipulado na sua programação.


Acordeão

Múltiplos painéis de expansão podem ser combinados em um acordeão. A entrada < multi="true" permite que o estado de expansões seja definido independentemente um do outro. Quando < multi="false" (padrão), apenas um painel pode ser expandido em determinado momento.

Renderização preguiçosa

Por padrão, o conteúdo do painel de expansão será inicializado mesmo quando o painel estiver fechado. Para adiar a incialização até o painel ser aberto, o conteúdo deve ser fornecido como ng-template.

Card

A tag < mat-card> é um container de conteúdo para textos, fotos e ações no contexto de um único conteúdo.

Seções básicas do card

A placa mais básica precisa apenas de um elemento < mat-card> com algum conteúdo. No entanto, o Angular Material fornece várias seções predefinidas que você pode usar dentro de um < mat-card>.

  • < mat-card-title>: Título do cartão
  • < mat-card-subtitle>: Legenda do cartão
  • < mat-card-content>: Conteúdo do cartão principal, destinado a blocos de texto
  • < img-mat-card-image>: Imagem do cartão. Estica a imagem até a largura do container
  • < mat-card-actions>: Espaço para botões na parte inferior do cartão
  • < mat-card-footer>: Seção ancorada na parte inferior do cartão

    • Cabeçalhos do card

    Além das seções mencionadas acima, < mat-card-header> dá a capacidade de adicionar um cabeçalho rico ao card. Este cabeçalho pode conter:

  • < mat-card-title>: Um título dentro do cabeçalho
  • < mat-card-subtitle>: Uma legenda no cabeçalho
  • < img-mat-card-avatar>: Uma imagem usada como um avatar dentro do cabeçalho

    • Botão

    Os botões do Angular Material são elementos nativos < button> ou < a> aprimorados com o estilo de design do Material.


    Os elementos nativos < button> e < a> são sempre usados para fornecer a experiência mais direta e acessível para os usuários. Um elemento< button> deve ser usado sempre que alguma ação for executada. Um elemento < a> deve ser usado sempre que o usuário navegar para outra visão.

    Existem várias variantes de botão, cada uma aplicada com um atributo:

  • mat-button: Botão de texto retangular sem elevação
  • mat-raised-button: Botão retangular contido c/ elevação
  • mat-flat-button: Botão retangular contido s/ elevação
  • mat-stroked-button: Botão delineado retangular s/ elevação
  • mat-icon-button: Botão circular com um fundo transparente, destinado a conter um ícone
  • mat-fab: Botão circular com elevação, padrão para a cor de destaque do tema
  • mat-mini-fab: O mesmo que mat-fab, menor

    • Ícone

    O < mat-icon> facilita o uso de ícones baseados em vetores no seu aplicativo. Esta diretiva suporta fontes de ícone e ícones SVG, mas não formatos baseados em bitmap (png, jpg, etc.).

    Registrando ícones

    MatIconRegistry é um serviço injetável que permite associar nomes de ícones a URLs, SVGs, strings HTML e definiar aliases para classes de fontes CSS. seus métodos são discutidos abaixo.

    Ícones de fonte com ligaduras

    Algumas fontes são projetadas para mostrar ícones usando ligaduras , por exemplo, renderizando o texto "home" como uma imagem inicial. Para usar um ícone de ligadura, coloque seu texto no conteúdo do componente mat-icon.

    por padrão mat-icon espera a fonte de ícones do Material. Você pode especificar uma fonte diferente configurando a entrada fontSet para a classe CSS a ser aplicada para usar a fonte desejada ou para um alias anteriormente registrado em MatIconRegistry.registerFontClassAlias.


    Ícones com fonte CSS

    As fontes também podem exibir ícones definindo uma classe CSS para cada glifo de ícone, que normalmente usa um seletor :before para fazer com que o ícone apareça. A FontAwesome usa essa abordagem para exibir seus ícones. Para usar essa fonte, defina a entrada fontSet como a classe CSS da fonte (a própria classe ou um alias registrado) e defina a entrada fontIcon para a classe do ícone específico a ser exibido.

    Para ambos os tipos de fontes de ícones, você precisa especificar a classe de fonte padrão a ser usada quando fontSet não estiver definida.
