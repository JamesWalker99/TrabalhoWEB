Checkbox

< mat-checkbox> fornece a mesma funcionalidade que um nativo < input type="checkbox> aprimorado com os estilos e animações do Material Design.

Clique!

Etiqueta da caixa de seleção

O rótulo da caixa de seleção é fornecido como o conteúdo para o elemento < mat-checkbox> . O label pode ser posicionado antes ou depois da caixa de seleção definindo a propriedade labelPosition para 'before' ou 'after'.

Se, por acaso, você não quiser que o label apareça ao lado da caixa de seleção, poderá usar aria-label ou aria-labelledby para especificar um rótulo apropriado.

Além disso, o < mat-checkbox> é compatível com o @angular/forms e suporta tanto o FormsModule quanto o ReactiveFormsModule.


Estado indeterminado

O < mat-checkbox> possui um estado indeterminate semelhante ao < input type="checkbox>. Embora a propriedade indeterminate da caixa de seleção seja verdadeira, ela será renderizada como indeterminada, independentemente do valor do checked e, qualquer interação com a caixa de seleção por um usuário (isto é, clicando) removerá o estado indeterminado.


Configuração de ação

Quando o usuário clica no < mat-checkbox> , o comportamento padrão é alternar o valor de checked e definir indeterminate como false. Este comportamento pode ser personalizado fornecendo um novo valor de MAT_CHECKBOX_CLICK_ACTION para a caixa de seleção.


Os valores possíveis são:


noop

Não altera o valor do checked ou de indeterminate. Os desenvolvedores possuem o poder de implementar ações de clique personalizadas.


check

Altera o valor de checked, ignorando o valor indeterminate. Se a caixa de seleção estiver no estado indeterminate, a caixa de seleção será exibida como uma caixa de seleção indeterminate, independente do valor de checked.


check-indeterminate

É o comportamento padrão do < mat-checkbox> . Com o indeterminate sempre definido para false no clique do usuário. Isso corresponde ao comportamento nativo do < input type="checkbox>.

Input

O matInput é uma diretiva que permite que os elementos nativos de < input> e < textarea> possam trabalhar em < mat-form-field> .

Atributos

Todos os atributos que podem ser usados normalmente com o < input> podem ser usados como elementos internos em < mat-form-field> também. Isso inclui as diretivas angulares como o ngModel e formControl.

A única limitação é que o atributo type só pode ser um dos valores suportados por matInput.


Tipos suportados:
  • color
  • date
  • datetime-local
  • email
  • month
  • number
  • password
  • search
  • tel
  • text
  • time
  • url
  • week

    • Recursos do campo de formulário

    Existem vários recursos do < mat-form-field> que podem ser usados com qualquer < input matInput> . Estes incluem mensagem de erro, dicas, prefixo, sufixo e temas.


    placeholder

    O placeholder é o texto mostrado quando o rótulo < mat-form-field> está flutuando, mas a entrada está vazia. Ele é usado para fornecer ao usuário uma dica adicional sobre o que deve digitar na entrada. O placeholder pode ser especificado configurando o atributo placeholder e, em alguns casos, o < mat-form-field> pode usar o placeholder como rótulo.


    Mudança quando mensagens de erro são mostradas

    O < mat-form-field> permite associar mensagens de erro ao seu matInput. Por padrão, as mensagens de erro são mostradas quando o control é inválido e/ou o usuário interagiu de maneira errada. Se você deseja substituir esse comportamento (por exemplo, mostrar o erro assim que o control for inválido), você poderá usar a propriedade errorStateMatcher. A propriedade usa uma instância de um objeto errorStateMatcher. Um errorStateMatcher deve implementar um único método isErrorStage que leva o FormControl para o matInput, bem como o formulário pai retorna um valor booleano indicando se o valor deve ser mostrado ou não (true ou false).

    Um correspondente do estado global de erro pode ser especificado configurando o provedor errorStateMatcher. Isso se aplica a todas as entradas. Por conveniência, ShowOnDirtyErrorStateMatcher está disponível para mostrar quando a entrada é inválida.

    Select

    O < mat-select> é um controle de formulário usado para selecionar um valor de um conjunto de opções, semelhante ao elemento nativo < select> e ele é projetado para funcionar dentro de um elemento< mat-form-field>.

    Para adicionar opções ao select, adicione o elemento < mat-option> ao < mat-select>. Cada < mat-option> tem uma propriedade value que pode ser usada para definir o valor que será selecionado se o usuário escolher essa opção. O conteúdo de < mat-option> é que será mostrado para o usuário.

    O < mat-select> também suporta todas as diretivas de formulário a partir do núcleo FormsModule (NgModel) e ReactiveFormsModule (FormControl, FormGroup, etc). Tal como acontece no elemento nativo < select>, < mat-select> também suporta uma função compareWith.

    O < mat-select> apresenta formas singulares de apresentação que podem ser melhores consultadas no site do Material, tais como:

  • Desativar opções selecionadas ou individuais;
  • Redefinir o valor de seleção;
  • Criar grupos de opções;
  • Seleção múltipla;
  • Personalização do rótulo do gatilho;
  • Desativar efeito de ondulação;
  • Adicionar estilos personalizados ao painel suspenso;
  • Mudança quando mensagem de erro são mostradas;
  • Interação com o teclado;

    • Autocomplete

    O autocomplete é uma entrada de texto normal aprimorada por um painel de opções sugeridas.

    {{name}}

    Preenchimento automático simples

    Comece adicionando um matInput ao seu modelo. Vamos supor que você esteja usando a diretiva FormControl e ReactiveFormsModule para rastrear o valor da entrada.


    < mat-form-field>
    < input type="text" matInput [formControl]="myControl">
    < / mat-form-field>

    Em seguida, crie o painel de preenchimento automático e as opções exibidas dentro dele. Cada opção deve ser definida por uma tag mat-option. Defina a propriedade de valor de cada opçãoc como desejar na seleção da opção.


    < mat-autocomplete>
    < mat-option *ngFor="let option of options" [value]="option"> [ option ]
    < /mat-option>
    < /mat-autocomplete>

    Agora vamos precisar ligar a entrada de texto ao seu painel. Podemos fazer isso exportanto a instância do painel de preenchimento automático para uma variável de modelo local (aqui denominamos "auto") e vinculando essa variável à matAutoComplete propriedade da entrada.


    < mat-form-field>
    < input type="text" matInput [formControl]="myControl" [matAutocomplete]="auto">
    < /mat-form-field>
    < mat-autocomplete #auto="matAutocomplete">
    < mat-option *ngFor="let option of options" [value]="option">
    [[ option ]]
    < /mat-option>
    < /mat-autocomplete>


    Além disso, o autocomplete possui funções singulares que também podem ser consultadas no site do Material, tais como:

  • Adicionando um filtro personalizado;
  • Configurando valores de controle de exibição separados;
  • Realçando automaticamente a primeira opção;
  • Interação com o teclado;
